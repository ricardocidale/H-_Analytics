## TASK 2: Intelligence-First Auto-Trigger (FEATURE GAP)

**Current state:** When a user visits PropertyEdit or CompanyAssumptions for the first time, a `FirstVisitBanner` appears saying "Ask the Analyst." The user must click it manually.

**What needs to happen:** Auto-trigger research on first visit (no click required). The infrastructure is all there:
- `usePageVisit()` hook returns `isFirstVisit: boolean`
- `generateResearch()` function exists in PropertyEdit
- Server endpoints for tracking are wired

**Implementation — in `client/src/pages/PropertyEdit.tsx`:**

Add a `useEffect` that watches `isFirstVisit` and auto-triggers:

```tsx
// Auto-trigger research on first visit (intelligence-first flow)
useEffect(() => {
  if (isFirstVisit && !isGenerating && !intelligenceClicked && property?.id) {
    // Small delay so the page renders first
    const timer = setTimeout(() => {
      setIntelligenceClicked(true);
      generateResearch();
    }, 1500);
    return () => clearTimeout(timer);
  }
}, [isFirstVisit, isGenerating, intelligenceClicked, property?.id]);
```

Do the same for `CompanyAssumptions.tsx` if it has research generation.

**UX:** While auto-research runs, show a status animation (use the AI animations you built). Something like "The Analyst is reviewing your property..." with a subtle pulse.
