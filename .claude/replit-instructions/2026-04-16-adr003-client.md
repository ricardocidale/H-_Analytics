## TASK 1: Tab Badge Counts — "3 fields to review"

Each tab in Company Assumptions should show a badge with the number of fields The Analyst has flagged or that are outside range.

**API:** `GET /api/guidance?entityType=company&entityId={companyId}`
Returns array of `{ assumptionKey, valueLow, valueMid, valueHigh, confidence }`

**Logic:** For each tab, count guidance rows where the current field value is outside the range [valueLow, valueHigh]. Map `assumptionKey` to tabs:

| Tab | Fields |
|-----|--------|
| Setup | companyName, companyCountry, companyCity, depreciationYears |
| Funding | capitalRaise1Amount, capitalRaise2Amount, fundingInterestRate |
| Revenue | baseManagementFee, incentiveManagementFee, service categories |
| Compensation | staffSalary, partnerCompYear1-10, staffTier* |
| Overhead | officeLease, professionalServices, techInfrastructure, businessInsurance, travelCostPerClient, itLicensingPerClient, marketingRate, miscOperations |
| Tax & Exit | companyTaxRate, costOfEquity, exitCapRate, salesCommissionRate |
| Property Defaults | eventExpenseRate, otherExpenseRate, utilitiesVariableSplit |

**UI:** Show a small amber badge on the tab trigger: `Revenue (2)` or `Compensation ✓`

## TASK 2: Returning User Delta Banner

When a user returns after being away, show what changed.

**New endpoint needed:** `GET /api/changes-since?since={ISO timestamp}`
Server queries `assumption_change_log` for entries after the timestamp.
Returns: `{ count, changes: [{ entityType, fieldName, previousValue, newValue, changeSource, createdAt }] }`

**Client logic:**
1. On login/page load, get `lastVisitedAt` from `user_page_visits` for the dashboard
2. Call `/api/changes-since?since={lastVisitedAt}`
3. If `count > 0`, show banner: "The Analyst has {count} updates since your last visit"
4. Clicking the banner opens a panel showing the changes grouped by entity

## TASK 3: Company Setup Gate — Client Error Handling

The research endpoint returns `{ error, code: "COMPANY_SETUP_INCOMPLETE" }` if company name or start date are missing.
The export endpoint returns `{ error, code: "COMPANY_SETUP_INCOMPLETE" }` if company name is missing.
The export endpoint returns `{ error, code: "PROPERTIES_EXCLUDED" }` if properties were excluded by The Analyst.

**Handle these in the research trigger and export flow:**
- `COMPANY_SETUP_INCOMPLETE`: show toast "Company name and start date are required. Set them on the Company Assumptions page." with a "Go to Assumptions" button.
- `PROPERTIES_EXCLUDED`: show toast listing the excluded properties and why. Link to Admin for data fixes.

## TASK 4: Research Fitness Badges on Property Cards

Properties now have richer `validationStatus` values including `excluded_data` and `excluded_admin`.

**Update `ValidationStatusBadge` component:**

| Status | Badge | Color |
|--------|-------|-------|

| `validated` | "Research-ready" | Green |

**Admin panel:** In Admin > Properties, show the `validationReason` text and allow admin to set `excluded_admin`. Admin CANNOT set `excluded_data` properties back to `validated` — only fixing the data and re-running The Analyst does that.

## TASK 5: First Login Flow

On first login (no `user_page_visits` records for this user):

1. Dashboard shows a banner: "Welcome. Let's confirm your management company information."
2. Banner button navigates to `/company/assumptions?tab=setup`
3. After user saves Setup tab, the endorsed flag is set
4. Next visit to Dashboard: banner gone, Analyst first-pass results shown

Do NOT build a separate wizard/onboarding page. Just a smart banner on the Dashboard that goes away after Setup is endorsed.
