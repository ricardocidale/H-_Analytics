## Context

Child (a) widened `MgmtCoSpecialistConfigs.funding` to accept an optional `deps?: FundingSpecialistDeps`. This child:

1. Builds the deps bundle in the AnalystButton-triggered route handler — assembling `FundingOrchestratorAdapter`, `ComparablesFetcher`, `EngineClientDeps`, `cacheKeyArgsBuilder`, `contextResolver` from request-scoped state (user id, scenario id, engine version, portfolio context, persona).
2. Wires the bundle into `createMgmtCoRouter` for the Funding scope only (Revenue stays on legacy until G2).
3. Adds an integration test asserting the Tier-1 path produces a verdict with all three ADR-008 meta fields populated (`cognitiveRunId`, `vendorsUsed`, `cacheState`) and **no** `fallbackReason`.

**Trigger discipline (binding):** the new dispatch site MUST live only in code paths that fire on AnalystButton click — never in a `save-tab` handler, never in a `useEffect`, never on page load / focus / scenario load. `tests/proof/analyst-trigger-discipline.test.ts` (16 tests) must continue to pass.

## Tasks

### S1 — New module: `server/ai/specialists/build-funding-deps.ts`

- **Files:**
  - NEW `server/ai/specialists/build-funding-deps.ts`
- **Change:** Pure factory `buildFundingSpecialistDeps(ctx)` that returns a `FundingSpecialistDeps` instance. Inputs:
  - `ctx.userId: string`
  - `ctx.scenarioId: string | null`
  - `ctx.engineVersion: string` (from `server/ai/engine-version.ts`)
  - `ctx.companyInputs: GlobalAssumptions` (the resolved-with-defaults globalAssumptions row, after `applyFundingDefaultsOverlay`)
  - `ctx.entityId: string` (the active company entity id)
  - `ctx.persona: FundingPersonaContext` (resolved upstream — initially a single canonical persona; persona-resolution work is Phase 4 deferred)
  - `ctx.priorVerdicts: FundingPromptInputContext["priorVerdicts"]` (read from `verdict_cache` if present; empty array otherwise)
  - `ctx.portfolio: FundingPromptInputContext["portfolio"]` (computed from `storage.getProperties()`)

  Outputs the bundle:
  - `orchestrator`: instance of `FundingOrchestratorAdapter` from `server/ai/specialists/mgmt-co-funding-orchestrator-adapter.ts`
  - `comparablesFetcher`: instance of `ComparablesFetcher` from same module
  - `engineClientDeps`: an `EngineClientDeps` constructed from `getAnthropicClient()`, `getGeminiClient()`, `getOpenAIClient()` (per `server/ai/clients.ts`) — the cognitive engine multi-vendor wiring
  - `cacheKeyArgsBuilder: (payload) => ({ entityId, engineVersion, scenarioId, companyInputs, persona })` — closes over `ctx`; per-call appends `payload`-derived `fieldGroup`
  - `contextResolver: (payload) => ({ portfolio: ctx.portfolio, persona: ctx.persona, priorVerdicts: ctx.priorVerdicts })`

- **Affected dependency surfaces:** S-Analyst-Verdict, S-Cognitive-Cache
- **Cross-check invariants:**
  - Per `domain-boundaries.md`: this module lives in `server/ai/`, may NOT import from `client/` and may NOT import `db` directly (storage facade only).
  - Per `llm-vendor-roster.md` requirement 7: the `engineClientDeps` MUST wire models from at least two vendors. Anthropic + Gemini + OpenAI satisfies the floor.
  - Per `analyst-trigger-discipline.md`: the factory is pure — it does NOT itself call `consultCognitive` or dispatch. It only assembles the deps so the Specialist can.
- **Acceptance criteria:**
  - [ ] `tsc --noEmit` returns 0 errors.
  - [ ] Pure: factory has no side effects beyond constructing the object (no DB write, no LLM call, no cache write).
  - [ ] Vendor breadth: a unit test or assertion verifies the returned `engineClientDeps` exposes ≥2 vendors.
- **Test impact:** Add unit test `tests/analyst/specialists/build-funding-deps.test.ts` covering: (a) factory returns a complete `FundingSpecialistDeps`-shaped object; (b) `cacheKeyArgsBuilder` appends `fieldGroup` correctly per payload; (c) `contextResolver` returns frozen context (no leak of mutable references).
- **Rollback:** Delete the new file + test.

### S2 — Re-route `/api/analyst/refresh` Funding scope through `createMgmtCoRouter`

- **Files:**
  - `server/routes/analyst-admin.ts` (extend `analystRefreshHandler` ≈ lines 34-180)
- **Change:** Branch on the request scope. For `scope === "global-assumptions"` AND `fields` includes any Funding-scoped field (or `fields` is unset and we want to refresh Funding):
  1. Resolve `ctx` per S1's inputs (read `globalAssumptions` with overlay, read properties, build persona, read prior verdicts from cache).
  2. Call `buildFundingSpecialistDeps(ctx)` to get the bundle.
  3. Construct the router via `createMgmtCoRouter(deps, benchmarks, { configs: { funding: { deps: <bundle>, ...other configs } } })`.
  4. Dispatch `MGMT_CO_FUNDING_ID` via `router.dispatch(MGMT_CO_FUNDING_ID, payload)` where `payload` is the `CapitalRaiseInputs` derived from the saved-with-overlay `globalAssumptions`.
  5. Return the resulting `AnalystVerdict` JSON to the client (existing response shape; the `cooldown` / `requestId` / etc. envelope stays).

  For non-Funding scopes (Revenue today), keep the existing `runAnalystScoped` path. **Note in code comment** that Revenue gets its own deps factory in G2.

- **Affected dependency surfaces:** S-Route-AnalystRefresh, S-Analyst-Verdict, S-Cognitive-Cache
- **Cross-check invariants:**
  - Per `analyst-trigger-discipline.md`: the new dispatch site is ONLY inside `analystRefreshHandler` — which is the AnalystButton's route. No `useEffect` or `save-tab` in this packet.
  - Per `tests/proof/analyst-trigger-discipline.test.ts`: confirm the test's allowlist already includes `analyst-admin.ts` as a permitted dispatch site (it should — G1.5b-pre's proof test was scoped to `save-tab` paths).
  - Per `error-handling.md`: try/catch around the router dispatch; on failure, log `[ERROR] [analyst-refresh] Funding Tier-1 dispatch failed` with context and fall back to a structured error response (no raw stack to client).
- **Acceptance criteria:**
  - [ ] `tsc --noEmit` returns 0 errors.
  - [ ] `npm run lint` — 0 warnings on the file.
  - [ ] On a real dev-server click of AnalystButton on Funding, the request reaches the new branch (not `runAnalystScoped`).
