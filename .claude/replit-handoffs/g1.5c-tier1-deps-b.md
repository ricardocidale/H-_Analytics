## Context

`server/ai/specialists/mgmt-co-funding-orchestrator-adapter.ts` defines the **interface** `FundingOrchestratorAdapter` — `run(input, options) → Promise<FundingOrchestratorResult>`. The interface's docstring at line 185 says *"the concrete adapter that wraps `server/ai/research-orchestrator.ts` is wired by Replit's route-handler slice"* — but no concrete implementation exists today.

Without a concrete implementation, child -c (route slice) cannot construct a real `FundingSpecialistDeps` bundle, so the Tier-1 path silently falls back to Tier-0 in production even though the wiring compiles.

This packet builds the concrete adapter. Pure server-side: wraps `orchestrateResearch()` (the existing 591-line streaming async generator at `server/ai/research-orchestrator.ts:332`), consumes the panel + synthesis stream, maps to `RawVerdictDimension[]`, and returns the structured `FundingOrchestratorResult`.

After this lands, child -c can compose the full deps bundle and the AnalystButton-triggered Funding consult genuinely runs N+1 cross-vendor cognitive synthesis.

## Tasks

### S1 — Build `server/ai/specialists/mgmt-co-funding-orchestrator-impl.ts`

- **Files:**
  - NEW `server/ai/specialists/mgmt-co-funding-orchestrator-impl.ts`
- **Change:** Concrete `class MgmtCoFundingOrchestrator implements FundingOrchestratorAdapter`. The `run(input, options)` method:
  1. Translates `FundingPromptInput` → the input shape `orchestrateResearch()` expects (typically `ResearchParams` per `server/ai/research-prompt-builders.ts`).
  2. Iterates `orchestrateResearch(...)` collecting:
     - Per-panel results (quantitative panel + market panel) → captures `vendorsUsed` from each panel's `model` field.
     - Synthesis output → captures `cognitiveRunId` (string id from synthesis), `promptEngineerRunId` (from prompt-engineer pre-stage), structured guidance.
  3. Maps synthesis guidance → `RawVerdictDimension[]` using `mapInputsToDimensionInputs(...)` from `mgmt-co-funding-prompt-input-builder.ts` as the dimension reference.
  4. Computes `convergenceScore` from synthesis output (per `CONVERGENCE_THRESHOLD = 0.6` in the adapter file's existing constant).
  5. Returns `FundingOrchestratorResult { cognitiveRunId, promptEngineerRunId, dimensions, vendorsUsed, convergenceScore }`.
  6. Honors `options.regressCount` (per `OrchestratorRunOptions`) by threading the regress counter into the prompt-engineer pre-stage so re-framing happens on retry.
- **Constructor inputs:** Takes `EngineClientDeps` (for cache + guidance reads) plus the LLM client trio (`anthropicClient`, `geminiClient`, `openaiClient`) per `server/ai/clients.ts`. Constructor is the seam for vendor-breadth verification.
- **Affected dependency surfaces:** S-Cognitive-Engine, S-Analyst-Verdict, S-Vendor-Breadth
- **Cross-check invariants:**
  - Per `domain-boundaries.md`: this module lives in `server/ai/`, may NOT import from `client/` and may NOT import `db` directly (storage facade only).
  - Per `llm-vendor-roster.md` requirement 7: returned `vendorsUsed` MUST be ≥2 distinct vendors. Constructor-time validation of the client trio ensures this.
  - Per ADR-007 §1 step 7: the adapter MUST expose `regressCount` in the input contract so the Specialist body can re-invoke with regress=1 / regress=2 on quality-check failure.
  - Per `engine-version.ts`: any prompt change accompanies an `ENGINE_VERSION` bump.
- **Acceptance criteria:**
  - [ ] `tsc --noEmit` returns 0 errors.
  - [ ] No lint warnings on the new file.
  - [ ] No import of `db` directly; all data access through storage facade.
- **Test impact:** New unit test in S2 covers the schema-mapping logic with stubbed clients.
- **Rollback:** Delete the new file.
