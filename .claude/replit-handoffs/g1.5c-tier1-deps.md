## Context

The Funding Specialist's Tier-1 evaluation path already exists at the `createFundingSpecialist(benchmarks, options, deps?)` factory in `engine/analyst/surface/mgmt-co/funding-specialist.ts:435`. Today the registry binding at `engine/analyst/surface/mgmt-co/index.ts:203` calls it with **only 2 arguments** — no `deps`. Result: every dispatch falls back to Tier-0 immediately and emits `meta.fallbackReason: "tier1_unavailable"` (per ADR-008 / G1.5a).

This packet closes the gap. The Tier-1 deps bundle exists in pieces (`FundingOrchestratorAdapter`, `ComparablesFetcher`, `EngineClientDeps`, `buildFundingCacheKey`, `buildFundingPromptInput`); they just need to be assembled into a `FundingSpecialistDeps` instance and threaded through `createMgmtCoRouter` to the registration site, then triggered from the AnalystButton-only route handler.

After this packet ships:
- AnalystButton-triggered Funding evaluation runs the N+1 cognitive pipeline.
- Verdicts carry `meta.cognitiveRunId` (non-null), `meta.vendorsUsed.length >= 2`, `meta.cacheState ∈ {"miss", "hit"}` per ADR-008.
- G1 closes; G2 (Revenue graduation) becomes unblocked.

Owner: **Claude Code** (engine + server/ai + server/routes + tests + phases — every layer of this packet is non-UI per the 2026-04-27 `claude-replit-split.md` revision).
