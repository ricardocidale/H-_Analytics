
  Current State

  Regular PDFs — Client-side jsPDF + autotable. Look like Excel table dumps with minimal branding. Primitive manual line charts. Small Helvetica font. No logos.

  Premium PDFs — Server-side, tries Anthropic Agent Skills API (beta), falls back to Claude generating a JSON structure that's then rendered with... the same jsPDF +
  autotable. So even the "premium" path ends up looking similar.

  The core problem: jsPDF + autotable produces functional but ugly output. No amount of AI structuring fixes that — it's still rendering with a limited PDF library.

  Recommendation: HTML-to-PDF via Puppeteer/Playwright

  Instead of fighting jsPDF's limitations, render PDFs from HTML/CSS — the same technology that makes the app look premium on screen.

  How it works:
  1. Build the PDF content as a styled HTML page (using the app's existing Tailwind classes, glassmorphism, branded colors)
  2. Use a headless browser (Puppeteer or Playwright) to render it to PDF
  3. The PDF looks exactly like the app — because it IS the app, printed

  Why this is better:
  - Full CSS support (gradients, shadows, custom fonts, flexbox, grid)
  - Charts render perfectly (Recharts → SVG → PDF)
  - Company logo, branded headers/footers
  - Consistent with what users see on screen
  - No more primitive line drawing or autotable theming

  What about Replit? Puppeteer works on Replit (Chromium is available). Alternatively, there are lighter options like @react-pdf/renderer which renders React components to
  PDF without a browser.
