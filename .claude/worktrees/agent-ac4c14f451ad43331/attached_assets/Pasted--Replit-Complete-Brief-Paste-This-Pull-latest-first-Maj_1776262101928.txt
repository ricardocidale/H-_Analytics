### 1. User Activity Tracking Database

Build a `user_page_visits` table to track every user's interaction with every page. Rolling 12 months.

**Schema:**
```sql
CREATE TABLE user_page_visits (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  page_key TEXT NOT NULL,
  entity_type TEXT,
  entity_id INTEGER,
  first_visited_at TIMESTAMP DEFAULT NOW(),
  last_visited_at TIMESTAMP DEFAULT NOW(),
  last_saved_at TIMESTAMP,
  last_analyst_run_at TIMESTAMP,
  endorsed BOOLEAN DEFAULT FALSE,
  compulsory_fields_complete BOOLEAN DEFAULT FALSE,
  visit_count INTEGER DEFAULT 1,
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(user_id, page_key)
);
CREATE INDEX idx_user_page_visits_user ON user_page_visits(user_id);
CREATE INDEX idx_user_page_visits_page ON user_page_visits(page_key);
```

**page_key format:** `"property:{id}:assumptions"`, `"property:{id}:edit"`, `"company:assumptions"`, `"global:settings"`

**Behavior:**
- On page load: upsert visit record, increment visit_count, update last_visited_at
- On Save: set `endorsed = true`, `last_saved_at = NOW()`
- On Ask the Analyst completion: set `last_analyst_run_at = NOW()`, then compulsory save
- On leaving without saving: auto-save, set `endorsed = true`
- If compulsory fields not complete and user leaves: treat as non-visit, don't set endorsed
- If intelligence is old (>24h): show glowing "Ask the Analyst" button, nudge to regenerate
- Auto-cleanup: delete rows older than 12 months (add to ambient scheduler)

**API endpoints (all requireAuth, scoped to current user):**
- `GET /api/page-visit/:pageKey`
- `POST /api/page-visit/:pageKey/visit`
- `POST /api/page-visit/:pageKey/save`
- `POST /api/page-visit/:pageKey/analyst-run`

Add Drizzle schema to `shared/schema/`, storage class, and routes.

### 4. Intelligence-First Page Behavior

Every page with inputs/assumptions must follow these rules:

**First visit (endorsed = false):**
1. Pre-fill fields with defaults (from property defaults, country defaults, benchmarks)
2. Show glowing "Ask the Analyst" button with NeuralGlow animation
3. Show banner: "The Analyst hasn't reviewed these assumptions yet"
4. Block downstream fields until compulsory fields are completed
5. After Ask the Analyst completes → compulsory save → set endorsed = true

**Return visit (endorsed = true, intelligence fresh):**
1. Show saved values — never overwrite
2. Analyst Notes visible next to fields
3. Status bar: "Up to date"
4. "Ask the Analyst" button available but not glowing

**Return visit (endorsed = true, intelligence old >24h or assumptions changed):**
1. Show saved values
2. Status bar: "Due for review" with glowing "Ask the Analyst" button
3. Nudge but don't block — user can still edit and save

**Leaving without saving:**
1. Show reminder: "You have unsaved changes"
2. If user dismisses → auto-save current values
3. Set endorsed = true
