  === Meta: the --no-verify problem ===                                                           
                                                                                                  
  The retro Phase 2c packet documents that every Phase 2c/3 commit used                           
  --no-verify because the pre-commit hook times out (>120s). "Manually
  verified" is weaker than hook-verified — it skips whatever the hook does                        
  automatically (e.g. formatter, secret scan, staged-files-only tests). The                       
  hook timeout is a real bug, not a permanent workaround.                                         
                                                                                                  
  Two requests:                                                                                   
                                                                                                  
    (a) Before the next commit on this branch, spend 30 min profiling the                         
        hook. Most likely cause: vocab test or verify:summary being run
        in-hook when they should only run in CI. Fix the hook config                              
        (lint-staged scope, husky _/pre-commit script) so the hook runs                           
        under 30s. If you can't fix in 30 min, file BLOCKED-precommit-hook.md                     
        so it's visible instead of buried in a retro packet.                                      
                                                                                                  
    (b) Until the hook is fixed, stop using --no-verify and instead invoke                        
        the 5 gates explicitly in order per .claude/rules/pre-commit-                             
        verification.md. That forces the same discipline without the                              
        timeout. Commit messages should still carry the Verified: line.
