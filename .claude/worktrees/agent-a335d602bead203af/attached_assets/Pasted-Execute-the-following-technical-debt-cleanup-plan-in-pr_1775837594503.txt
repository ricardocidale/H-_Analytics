## Task 1: Hardcoded Colors Cleanup (68 across 11 files)

The March 20, 2026 session audit found 68 hardcoded hex colors across 11 files that should use CSS variable tokens instead. The project rule (rules/design-standards.md) says "No raw hex in components — use CSS variable tokens."

Find all hardcoded hex colors (#xxxxxx or #xxx patterns) in client/src/ components (NOT in theme definitions, CSS files, or tailwind config — those are where colors are DEFINED). Replace them with the appropriate CSS variables from the theme system. Load `.claude/skills/design-system/SKILL.md` and `.claude/skills/ui/theme-engine.md` for the correct token names.

Common mappings:
- Brand/primary colors → `hsl(var(--primary))` or `text-primary`
- Background colors → `hsl(var(--background))`, `hsl(var(--card))`, `hsl(var(--muted))`
- Text colors → `hsl(var(--foreground))`, `hsl(var(--muted-foreground))`
- Border colors → `hsl(var(--border))`
- Accent colors → `hsl(var(--accent))`
- Chart colors → `hsl(var(--chart-1))` through `hsl(var(--chart-5))`

IMPORTANT: Do NOT change colors in:
- Theme definition files (where CSS variables are set)
- Tailwind config
- Export/PDF generation code (those resolve theme colors via `resolveThemeColors()`)
- RichBlockRenderers.tsx (those are H+ design system constants — navy #112548, teal #0091AE, gold #FDB817)
- Test files
