#295 - Allow custom city entry in address fields
What & Why
The city dropdown in property and company address forms only allows selecting from the country-state-city npm package's static dataset. Many smaller towns, hamlets, and CDPs (like Loch Sheldrake, NY) are missing from this dataset, preventing users from entering valid addresses. The city field should allow users to either pick from the existing list or type in a custom city name.

Done looks like
On both the property address form and the company address form, the city field is a searchable combobox (typeahead) instead of a strict dropdown
Users can type a city name to filter the existing list
If the desired city isn't in the list, users can type and submit a custom city name (free-text entry)
Previously saved cities (including custom ones) display correctly when revisiting the form
The address "59 Hazelnis Dr, Loch Sheldrake, NY 12759" can be fully entered
Out of scope
Changing the underlying country-state-city data source or adding cities to it
Geocoding or validating custom city entries
Changing the country or state fields (those remain strict dropdowns)
Tasks
Replace the city <Select> with a combobox component that supports both selecting from the list and typing a custom value. If a suitable combobox already exists in the UI library, use it; otherwise build a minimal one.
Update BasicInfoSection.tsx (property address) to use the new combobox for the city field.
Update CompanySetupSection.tsx (company address) to use the same combobox for the city field.
Ensure handleCityChange in use-geo.ts works correctly with free-text city values (no code changes needed if it already just passes the string through).
Verify that a previously-saved custom city name loads and displays correctly when the form is reopened.
