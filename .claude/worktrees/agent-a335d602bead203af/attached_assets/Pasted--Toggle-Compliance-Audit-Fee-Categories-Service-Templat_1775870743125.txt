  ---                                                                                                                                                                                                                  
  Bug 1 Fixed (CRITICAL): Server compute path omitted service templates
                                                                                                                                                                                                                       
  Files changed: server/finance/service.ts, server/routes/finance.ts
                                                                                                                                                                                                                       
  What was wrong: When USE_SERVER_COMPUTE=true (the default deployment mode), the server-side compute functions computePortfolioProjectionWithAudit and computeCompanyProjection called                                
  generateCompanyProForma(properties, globalAssumptions, months) with only 3 arguments. The 4th parameter serviceTemplates was never passed. This meant:                                                               
  - The company engine's hasServiceTemplates gate was always false                                                                                                                                                     
  - computeCostOfServices was never called                        
  - Vendor cost-of-services was always zero
  - Company gross profit was overstated by the entire vendor cost amount                                                                                                                                               
  
  What was fixed:                                                                                                                                                                                                      
  - Added serviceTemplates?: ServiceTemplate[] to both ComputePortfolioInput and ComputeCompanyInput interfaces in server/finance/service.ts
  - Both generateCompanyProForma calls in that file now pass input.serviceTemplates                                                                                                                                    
  - Both finance routes (/api/finance/compute and /api/finance/company in server/routes/finance.ts) now load service templates from storage via storage.getAllServiceTemplates(), map them to the ServiceTemplate type,
   and include them in the compute input                                                                                                                                                                               
  - The engine already accepts templates as an optional 4th arg, so no engine changes were needed                                                                                                                      
                                                                                                                                                                                                                       
  Impact: Company financial statements now correctly include vendor cost-of-services when service templates are configured with serviceModel: 'centralized'. This affects the Company page, consolidated exports, and  
  any downstream analysis that uses company P&L data.                                                                                                                                                                  
                                                                                                                                                                                                                       
  ---                                                                                                                                                                                                                  
  Bug 2 Fixed (HIGH): Service template admin routes missing cache invalidation

  File changed: server/routes/admin/services.ts

  What was wrong: All 4 mutation handlers (create, update/toggle isActive, delete, sync-to-properties) did NOT call invalidateComputeCache(). When an admin toggled a service template inactive or changed its markup  
  rate, the server-side compute cache continued serving stale company financials until some other event (like a property update) happened to invalidate it.
                                                                                                                                                                                                                       
  What was fixed: Added import { invalidateComputeCache } from "../../finance/cache" and invalidateComputeCache() call in each of the 4 mutation handlers: POST create, PATCH update, DELETE, and POST sync.

  3. Service Templates Not in Scenario Snapshots                                                                                                                                                                       
                                                                                                                                                                                                                       
  Context (informational, discuss before implementing): Service templates are NOT included in scenario snapshots. The scenarios table has no serviceTemplates column, and buildCreateSnapshotData does not capture     
  them. This means:                                         
  - Saving a scenario captures property data and fee categories, but NOT the company's service template configuration                                                                                                  
  - Loading a scenario restores properties and fees but uses CURRENT service templates                                                                                                                                 
  - If templates changed between save and load, the company P&L will differ from what was originally saved
                                                                                                                                                                                                                       
  This may be by design (service templates are company infrastructure, not scenario-specific). But if users expect scenario snapshots to fully reproduce financial results, this is a gap.                             
                                                                                                                                                                                                                       
  Decision needed: Should service templates be included in scenario snapshots? If yes, add a serviceTemplates JSONB column to the scenarios table, capture them in buildCreateSnapshotData, and restore them on load.  
  If no, document that company service template configuration is not scenario-specific.
