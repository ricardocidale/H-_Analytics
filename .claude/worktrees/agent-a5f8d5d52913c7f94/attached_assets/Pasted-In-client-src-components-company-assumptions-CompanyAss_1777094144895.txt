Why Radix silently produces the empty state
When the value passed to <Tabs> matches no TabsTrigger and no TabsContent, Radix does not throw — it simply renders no active trigger (data-state="inactive" on every one) and no content. The capability TabsContent are additionally wrapped in {tabsList.find((t) => t.value === "X") && …} guards (e.g. line 161), so a stale capability key also drops the content from the tree entirely. The workflow / identity / sources TabsContent are unguarded but don’t match the stale value either, so none of them render.
