HIGH FINDINGS (4)
================================================================================

H1. xlsx package v0.18.5 — known vulnerabilities
   Location: package.json:154
   Details: The SheetJS xlsx package has known prototype pollution and
            arbitrary code execution vulnerabilities in older versions.
            Version 0.18.5 is the community fork.
   Fix: Run npm audit, upgrade or replace.

H2. any type proliferation (338 total across codebase)
   Locations:
     - client/src/lib/exports/companyExports.ts (61 any)
     - server/routes/admin/marcela.ts (40 any)
     - client/src/lib/exports/researchPdfExport.ts (27 any)
     - client/src/lib/exports/excel/property-sheets.ts (21 any)
     - server/routes/admin/tools.ts (21 any)
     - client/src/lib/exports/excel/portfolio-sheet.ts (18 any)
     - server/integrations/document-ai.ts (11 any)
   Details: ESLint any ban only covers calc/ and engine/, not server/ or
            client/lib/exports/ where the worst offenders live. Function
            signatures like data: { years: number[]; rows: any[] } defeat
            TypeScript guarantees on financial export data shapes.
   Fix: Extend ESLint any ban to all source files, type the export row builders.

H3. Math.pow in financial calculation paths
   Locations:
     - analytics/returns/irr.ts:13,16,28,78,110 — IRR solver (5 occurrences)
     - domain/types/rounding.ts:8 — most-called financial function in system
     - server/calculation-checker/index.ts:179 — CAGR computation
   Details: All violate the project's own financial-safety.md rules which
            mandate dPow from calc/shared/decimal.ts for decimal-safe
            exponentiation. The IRR solver with large exponents (t up to 120
            for monthly, periodsPerYear=12) introduces floating-point error
            that compounds through Newton-Raphson iterations.
   Fix: Replace all with dPow from calc/shared/decimal.ts (7 occurrences).

H4. CI pipeline missing build step and e2e tests
   Location: .github/workflows/ci.yml
   Details: Runs unit tests only. No npm run build, no e2e tests, no coverage
            reporting. e2e smoke test exists but is permanently skipped in CI.
   Fix: Add build verification and coverage thresholds to CI.

MEDIUM FINDINGS (10)
================================================================================

M1. PII in seed-users.json (repo is public!)
   Location: server/seed-users.json
   Details: Contains 8 real personal email addresses (e.g.,
            ricardo.cidale@norfolkgroup.io, wlaruffa@gmail.com,
            lemazniku@icloud.com, kit@kitcapital.com) along with full names,
            companies, and titles. Since the repo is public on GitHub, anyone
            can see this data.
   Fix: Move to env vars or a non-committed config file.

M2. dDiv silently returns 0 on division by zero
   Location: calc/shared/decimal.ts:19
   Details: The comment says "callers pre-guard zero divisors" but this is
            silent zero-coercion that contradicts financial-safety.md rule #3.
            If a denominator is unexpectedly zero (e.g., missing input),
            dDiv(100000, 0) returns 0 instead of failing fast, potentially
            masking data corruption.
   Fix: Add assertFinite or explicit error handling per project rules.

M3. FCF working capital permanently hardcoded to zero
   Location: analytics/fcf/compute-fcf.ts:99
   Details: const working_capital_change = 0; The MonthlyFinancials interface
            includes accountsReceivable, accountsPayable, and
            workingCapitalChange fields, but the FCF module ignores them.
            FCFF is overstated when working capital consumes cash (growing
            revenue) and understated when it releases cash (declining revenue).
   Fix: Implement FCF working capital from MonthlyFinancials data.

M4. Trial balance accumulation uses += instead of dSum
   Location: engine/posting/trial-balance.ts:25-26
   Details: Both buildTrialBalance and buildCumulativeTrialBalance accumulate
            debits/credits via running += loops. financial-safety.md rule #6
            states: "Running += loops on financial totals must use
            dSum(parts) to avoid floating-point drift." For properties with
            hundreds of entries, cumulative drift could produce penny
            discrepancies that fail reconciliation.
   Fix: Collect parts in array, use dSum once.

M5. Balance sheet tolerance hardcoded at 0.01
   Location: statements/balance-sheet.ts:73, statements/reconcile.ts:13
   Details: const TOLERANCE = 0.01 is hardcoded rather than derived from the
            rounding policy's precision. For precision:0 (whole dollars), the
            tolerance should be larger. For precision:4, it could be tighter.
   Fix: Derive tolerance from rounding policy or use withinTolerance() from
        calc/shared/utils.ts.

M6. Incomplete SSRF blocklist
   Location: server/routes/uploads.ts:210-214
   Details: When processing photos from URLs, a blocklist prevents SSRF to
            metadata endpoints. Missing coverage for: IPv6 mapped addresses
            (::ffff:127.0.0.1), DNS rebinding attacks, private IP ranges
            (10.x.x.x, 172.16-31.x.x, 192.168.x.x).
   Fix: Expand blocklist with private IP ranges and IPv6 variants.

M7. Legacy Zustand store with stale hardcoded data
   Location: client/src/lib/store.ts
   Details: 170 lines of hardcoded property data for 5 properties (Hudson
            Estate, Eden Summit, Austin Hillside, Casa Medellin, Blue Ridge
            Manor). TanStack Query is the primary data source. The store
            duplicates types from shared/schema.ts and contains stale initial
            data that is never in sync with DB. Only imported by 11 files.
   Fix: Migrate remaining consumers to TanStack Query, slim store to UI-only state.

M8. Screenshot/test files bloating repo
   Location: Root directory (15+ PNG files, test-results.json 756KB,
             test-results-golden.json 290KB)
   Details: Development screenshots and auto-generated test result files add
            several MB to the repo without production value.
   Fix: Add test-results*.json and root *.png to .gitignore.

M9. Marcela tools timing-safe comparison leaks secret length
   Location: server/routes/marcela-tools.ts:20
   Details: The timingSafeEqual check includes a.length === b.length guard
            that leaks the secret length if buffers differ in size.
   Fix: Pad or hash both values to fixed length before comparison.

M10. No React component tests, no server route tests
    Location: tests/
    Details: 170 test files covering financial calcs thoroughly, but zero
             component tests and zero route handler tests. Significant blind
             spots in the test suite.
    Fix: Add component tests (React Testing Library) and route integration tests.

LOW FINDINGS (9)
================================================================================

L1. No explicit CSRF tokens
   Location: server/auth.ts
   Details: Mitigated by SameSite=Lax cookies and JSON content type, which
            prevents cross-origin form submissions. Risk is low but not zero.

L2. CSP allows unsafe-inline for scripts
   Location: server/index.ts:65
   Details: script-src 'self' 'unsafe-inline' and style-src 'self'
            'unsafe-inline' weakens CSP against script injection.
   Fix: Consider implementing nonce-based CSP for scripts.

L3. Dead path aliases @engine/ and @statements/
   Location: tsconfig.json:18-19, vite.config.ts:34
   Details: These aliases are configured but resolve to directories that do
            not contain the expected barrel exports.
   Fix: Remove dead aliases or ensure proper barrel exports.

L4. catch (error: any) pattern (~15 instances)
   Location: server/routes/admin/marcela.ts
   Details: Should use catch (error: unknown) with proper type narrowing.

L5. MIRR returns NaN instead of null (inconsistent with IRR)
   Location: calc/returns/mirr.ts:50
   Details: IRR solver returns null for invalid results; MIRR returns NaN.
            Inconsistent convention could cause downstream arithmetic issues
            if consumers don't check is_valid.
   Fix: Change MIRR to return null for consistency.

L6. Break-even margin_of_safety always returns zeros
   Location: calc/analysis/break-even.ts:181-184
   Details: The margin_of_safety output is defined in the interface but always
            returns { current_occupancy_cushion: 0, revenue_cushion_pct: 0 }.
            This is a stub — the function takes no current_occupancy input.
   Fix: Implement actual computation or remove from interface.

L7. DEPRECIATION_YEARS docs say 27.5, code uses 39 (code is correct)
   Location: CLAUDE.md vs shared/constants.ts:164
   Details: Code correctly uses 39 per IRS rules for nonresidential real
            property (hotels, IRC section 168(e)(2)(A)). CLAUDE.md reference
            is outdated.
   Fix: Update CLAUDE.md to say 39, not 27.5.

L8. Large dependency surface (~100+ direct deps)
   Location: package.json
   Details: Includes multiple AI SDK clients (Anthropic, OpenAI, Google,
            ElevenLabs, Perplexity), Twilio, Replicate, puppeteer-core.
            Each represents an attack surface.
   Fix: Periodic npm audit runs.

L9. Portfolio PDFs in public repo
   Location: Root directory
   Details: Intentional spec/reference docs for the financial platform, but
            visible to anyone since the repo is public. Not a code issue —
            informational only.
