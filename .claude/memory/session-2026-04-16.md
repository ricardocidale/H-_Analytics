## Key Decisions Made

1. **The HMC does NOT buy properties.** Property owners hire the HMC. This is permanent. See `.claude/memory/project_business_model_correction.md`.

2. **Defaults ≠ Assumptions.** Defaults are admin (Admin section). Assumptions are users (front of app). Company Assumptions page is user-facing.

3. **Save is per tab.** Each tab in Company Assumptions has its own Save. The Analyst validates after each save.

4. **The Analyst runs on whatever data exists.** It validates seed data, flags issues, and tells the user. It doesn't wait for permission. The only gate: companyName and modelStartDate must exist.

5. **The Analyst controls its own inputs.** Properties with bad data get `excluded_data` status. Admin cannot override — only fixing the data restores research fitness.

6. **Per-tab workflow.** User opens tab → sees seed data with Analyst ranges → edits what they know → saves → Analyst validates → flags issues → user adjusts or keeps → moves to next tab.
