## Context

The Funding Specialist's `candidateFields` references 5 fields that **don't exist in the user's cascade today**:

| Field | Schema column | Default seed | Funding-tab input | Form-hook wire |
|---|---|---|---|---|
| `runwayBufferMonths` | ❌ missing | ❌ missing | ❌ missing | hardcoded `null` |
| `sizingOvershootPct` | ❌ missing | ❌ missing | ❌ missing | hardcoded `null` |
| `trancheGapMonths` | ❌ missing | ❌ missing | ❌ missing | computed from dates |
| `revenueRampDelayMonths` | ❌ missing | ❌ missing | ❌ missing | hardcoded `null` |
| `burnFlexDownPct` | ❌ missing | ❌ missing | ❌ missing | hardcoded `null` |

They live ONLY as benchmark ranges in `capital_raise_benchmarks` (low/mid/high). The Specialist evaluates against benchmarks but the user has no surface to set their actual value. Per Ricardo's binding cascade rule, this is a violation.

This packet wires all four cascade surfaces. Once shipped, the Funding Specialist's `requiredFields` check passes against real user inputs (vs the `null` workaround), and Ricardo's three-tier discipline is honored.

Owner: **Replit** (UI + DB + Admin lane). Two-track when CC slice arises (none expected for this packet).
