  What was done (Claude Code)
                                                                                                                                                                                                                       
  The isActive toggle ("Included in portfolio" / "Excluded from portfolio") was audited across the entire codebase. Four bugs and one missing guard were found and fixed:                                              
                                                                                                                                                                                                                       
  Bugs fixed:                                                                                                                                                                                                          
  1. client/src/pages/Company.tsx — Client-side fallback path passed ALL properties (including inactive) to generateCompanyProForma and generatePropertyProForma. Inactive properties were inflating management company
   fee revenue. Added .filter(p => p.isActive !== false) before both engine calls.                                                                                                                                     
  2. client/src/pages/FundingPredictor.tsx — Same bug. Inactive properties inflated company revenue in funding analysis, causing overly optimistic cash runway projections. Added filter.
  3. server/routes/finance.ts — The /api/finance/compute and /api/finance/company endpoints trusted the client to pre-filter. Added defense-in-depth server-side filter on both routes so even direct API callers can't
   send inactive properties to the engine.                                                                                                                                                                             
  4. server/routes/calculations.ts — The verification checker (/api/verification/run) was calling storage.getAllProperties() and passing everything to the checker including inactive properties. Now filters to       
  active-only before verification.                                                                                                                                                                                     
                                                                                                                                                                                                                       
  Guard added:                                              
  5. server/notifications/engine.ts — evaluateAlertRules() had no check for property.isActive. If a user toggled a property inactive and then an update triggered alerts, they'd get confusing notifications about an  
  excluded property. Added if (property.isActive === false) return; at function entry.                                                                                                                                 
   
  Proof tests added:                                                                                                                                                                                                   
  - tests/proof/isActive-compliance.test.ts — 15 tests that verify every compute path (client and server), every server route, the checker, notifications, and scenario persistence all honor the isActive flag. These
  catch regressions if future code bypasses the filter.

  1. Company Assumptions — Fee Summary Table                

  File: client/src/pages/CompanyAssumptions.tsx line ~68, client/src/components/company-assumptions/PropertyFeeSummaryTable.tsx                                                                                        
   
  Problem: The "Per-Property Fee Summary" table at the bottom of the Company Assumptions page shows ALL properties including inactive ones. Since inactive properties are excluded from all financial calculations,    
  showing their fee rates here is misleading — users might think those fees are contributing to the management company P&L when they're not.
                                                                                                                                                                                                                       
  Fix options (pick one):                                   
  - Option A: Filter out inactive properties entirely — properties.filter(p => p.isActive !== false) before passing to the table
  - Option B (preferred): Keep them in the table but visually distinguish them — grey the row, reduce opacity, and add an "Excluded" badge. This lets users see the full picture while making the exclusion obvious.   
                                                                                                                                                                                                                    
  If going with Option B, use the same visual pattern as PortfolioPropertyCard.tsx which already does opacity-60 saturate-50 for inactive properties.
