  Brief 1 — SlideDecksTab card thumbnails (quick visual win)
                                                                                               
  Fix property photos in the Slide Decks admin tab card thumbnails.
                                                                                               
  PROBLEM:                                                        
  `SlideRender` in                                                                             
  `artifacts/hospitality-business-portal/src/components/admin/SlideDecksTab.tsx` lines 102–178 
  renders a dark gradient card for every property. It never reads `property.imageUrl` even
  though that field is already in the data.                                                    
                                                                  
  DONE LOOKS LIKE:
  - Cards show the actual property photo as the card background
  - A semi-transparent dark overlay (~rgba(0,0,0,0.45)) sits between image and text so         
  name/location/meta stay readable                                                             
  - Cards with no imageUrl keep the current gradient (no regression)                           
  - Both `/api/media/...` static paths and `/api/property-photos/:id/image` paths resolve      
  correctly                                                                                    
                                                                                               
  IMPLEMENTATION:                                                                              
  In the `SlideRender` component, when `property.imageUrl` is truthy, render an <img> (or CSS
  background-image) filling the card behind all existing text/gradient layers. When falsy, keep
   the existing gradient unchanged.                               
                                                                                               
  SCOPE: Only `SlideDecksTab.tsx`. No changes to PPTX generator, PropertyDeck.tsx, or how      
  imageUrl is set on the DB.
