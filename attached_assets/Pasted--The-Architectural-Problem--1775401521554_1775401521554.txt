  The Architectural Problem                                                                                                                                                                                                               
   
  Current flow is backwards:                                                                                                                                                                                                              
                                                            
  13 APIs gather data → dump into LLM context → LLM formats/narrates → extract values
                                                                                                                                                                                                                                          
  The LLM is acting as a formatter, not a researcher. APIs drive the output; if they're unavailable, quality collapses. Single model means single perspective with false confidence — no way to know when it's guessing.                  
                                                                                                                                                                                                                                          
  ---                                                                                                                                                                                                                                     
  The Right Architecture: LLM-First with N+1 + API Anchoring
                                                                                                                                                                                                                                          
  Core insight: Claude Opus has internalized CBRE, JLL, HVS, PKF, STR Global, CoStar white papers, academic hospitality research — millions of pages. For "boutique hotel ADR in Medellin" or "cap rate for a luxury STR in Colombia," it
  knows more than most APIs, especially for emerging markets where data coverage is thin. APIs are authoritative for real-time numbers (today's FRED rate, live OTA prices) but not for market understanding.                             
                                                            
  ---                                                                                                                                                                                                                                     
  Three-Phase Pipeline                                      
                      
  Phase 1 — Parallel LLM Research (the N)
                                                                                                                                                                                                                                          
  Two models run independently and simultaneously, each with a different lens:                                                                                                                                                            
                                                                                                                                                                                                                                          
  ┌──────────────────┬─────────────────────────────────────────────────────────────────────────────────────────┬──────────────────────────────────────────────────────────────────────────────────────┐                                   
  │      Model       │                                          Role                                           │                                         Gets                                         │
  ├──────────────────┼─────────────────────────────────────────────────────────────────────────────────────────┼──────────────────────────────────────────────────────────────────────────────────────┤                                   
  │ Gemini 2.5 Flash │ Quantitative analyst — ADR ranges, occupancy curves, cost benchmarks, comp-set          │ Location + property spec + Perplexity web search + Pinecone similar-property history │
  ├──────────────────┼─────────────────────────────────────────────────────────────────────────────────────────┼──────────────────────────────────────────────────────────────────────────────────────┤                                   
  │ Claude Sonnet    │ Market analyst — risk factors, qualitative positioning, supply narrative, macro context │ Same inputs                                                                          │                                   
  └──────────────────┴─────────────────────────────────────────────────────────────────────────────────────────┴──────────────────────────────────────────────────────────────────────────────────────┘                                   
                                                                                                                                                                                                                                          
  Each produces a fully structured output independently. No coordination.                                                                                                                                                                 
                                                            
  Phase 2 — API Validation (anchoring)                                                                                                                                                                                                    
                                                            
  APIs run after the LLM pass to validate specific anchor points:                                                                                                                                                                         
  - FRED: Lock in exact current interest rates (LLMs should never estimate live rates)
  - Xotelo / RapidAPI comps: Check LLM ADR estimate against actual live OTA listings                                                                                                                                                      
  - World Bank: Confirm macro numbers for international markets                     
  - CoStar/STR benchmarks: Professional industry validation                                                                                                                                                                               
                                                            
  APIs flag where LLM estimates diverge from reality by more than a threshold (say 20%). They don't replace the estimate — they inform confidence.                                                                                        
                                                                                                                                                                                                                                          
  Phase 3 — Synthesis (the +1)                                                                                                                                                                                                            
                                                                                                                                                                                                                                          
  Claude Opus reads:                                                                                                                                                                                                                      
  - Model A's full structured output                        
  - Model B's full structured output
  - API validation results (what matched, what diverged, by how much)

  It produces:                                                                                                                                                                                                                            
  - Consensus values where both models agree (high confidence)
  - Reconciled ranges where they disagree (the disagreement becomes the confidence band — conservative/aggressive naturally emerges from model divergence)                                                                                
  - Flagged metrics where LLM estimates contradict API data — Opus explains why                                                                           
  - Source attribution per value: "both models + CoStar confirmed" vs. "LLM estimate, no API coverage"                                                                                                                                    
                                                                                                                                                                                                                                          
  ---                                                                                                                                                                                                                                     
  Pinecone's Role — The Memory Layer                                                                                                                                                                                                      
                                                                                                                                                                                                                                          
  This is what makes the system improve over time. Three namespaces:
                                                                                                                                                                                                                                          
  research-history — Every completed research result gets embedded and stored. When researching a new property, retrieve the 5 most similar past analyses (same city, similar property type, similar room count). The LLMs walk in knowing
   what the system already learned.                                                                                                                                                                                                       
                                                                                                                                                                                                                                          
  market-snapshots — API results get indexed here with timestamps. CoStar data for Medellin from last month doesn't require another API call — retrieve from Pinecone. API calls only fire when data is stale.                            
  
  knowledge-base — Current attached_assets docs, methodology, business model. Replace the ephemeral in-memory embeddings with persistent indexed vectors.                                                                                 
                                                            
  The compounding effect: Research #1 on a Medellin property gets indexed. Research #5 on a different Medellin property retrieves it, making the LLMs more grounded in what the system already confirmed. The system gets smarter with    
  each run without retraining.                              
                                                                                                                                                                                                                                          
  ---                                                       
  Why N+1 Specifically
                                                                                                                                                                                                                                          
  A single model produces an answer with uniform false confidence. The N+1 structure exposes where the model actually has uncertainty:
                                                                                                                                                                                                                                          
  - Both models say ADR $900–$1,100 → High confidence, tight range                                                                                                                                                                        
  - Model A says $700–$900, Model B says $1,000–$1,300 → Real uncertainty, surface it, widen the band, flag for human review                                                                                                              
  - Both models say cap rate 9%, CoStar says 7% → API contradiction, Opus explains the discrepancy (maybe LLMs are using pre-2024 data; CoStar has current transaction data)                                                              
                                                                                                                                                                                                                                          
  Disagreements aren't failures — they're signal.                                                                                                                                                                                         
                                                                                                                                                                                                                                          
  ---                                                       
  What Changes in Code
                                                                                                                                                                                                                                          
  1. PineconeService — embed, upsert, query across 3 namespaces; replaces knowledge-base.ts in-memory store
  2. ResearchOrchestrator — replaces single generateResearchWithToolsStream. Runs Phase 1 models in parallel (Promise.all), runs Phase 2 API validation, sends to Phase 3 Opus synthesizer                                                
  3. ApiValidator — thin layer that compares LLM output against available API data, returns concordance/discordance per metric                                                                                                            
  4. research.ts route — simpler; delegates to ResearchOrchestrator rather than managing the LLM loop itself                                                                                                                              
  5. Pinecone indexing on save — after research completes and validates, embed + store in research-history
