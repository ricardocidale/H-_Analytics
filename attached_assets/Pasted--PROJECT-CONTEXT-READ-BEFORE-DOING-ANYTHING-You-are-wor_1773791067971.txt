  ## CRITICAL RULES — NEVER VIOLATE THESE

  1. **Financial accuracy beats everything** — never compromise calculations for UI improvements
  2. **All buttons say "Save"** — never "Update"
  3. **All DB access via IStorage interface** — no route may import db directly
  4. **No hardcoded model names, tax rates, or financial constants** — all come from DB with named-constant fallbacks
  5. **onSuccess financial mutations must call invalidateAllFinancialQueries(queryClient)** — never hand-pick query
  keys
  6. **calculationChecker.ts must never import from the client engine** — independence is the point
  7. **All properties created with userId: null** — shared ownership, visible to all users
  8. **calc/ files must never import from server/** — tool purity
  9. **Admin tabs with onSaveStateChange must use ref-based save handlers** — never put draft state in useCallback
  deps
  10. **No raw hex colors in components** — use CSS variable tokens
  11. **All financial line items need a ? tooltip** (HelpTooltip or InfoTooltip)
  12. **Every page must be graphics-rich** — charts, animations, Framer Motion, skeleton loading states
  13. **Full-scope exports only** — exporting from any tab exports all statements, never just the current tab
  14. **File naming at save step** — never add a filename field to the export dialog
