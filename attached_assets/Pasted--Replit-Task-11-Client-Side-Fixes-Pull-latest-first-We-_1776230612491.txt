## Replit Task: 11 Client-Side Fixes

### Fix 1: Block javascript: URIs in Rebecca markdown
**File:** `client/src/components/rebecca/RebeccaMarkdown.tsx`
**Bug:** LLM-generated markdown can include `javascript:` links that execute code.
**Fix:** In the `<a>` renderer, whitelist only `http://` and `https://` schemes. Any other href should render as plain text, not a link.

### Fix 2: ErrorBoundary shows raw Error.message to users
**File:** `client/src/components/ErrorBoundary.tsx` (line ~49) and `FinancialErrorBoundary` (line ~172)
**Bug:** Raw JavaScript error messages (including internal paths, SQL details) are shown in the fallback UI.
**Fix:** Replace `{this.state.error?.message}` with a generic message: "Something went wrong. Please reload the page."

### Fix 3: RebeccaPanel stale closure
**File:** `client/src/components/rebecca/RebeccaPanel.tsx` (lines 169-193)
**Bug:** `useEffect` calls `loadConversation` and `sendAutoGreeting` but neither is in the dependency array. Can cause AI responses saved to the wrong conversation.
**Fix:** Add `loadConversation` and `sendAutoGreeting` to the `useEffect` dependency array, or wrap them in refs.

### Fix 4: poi.rating unescaped in map popup
**File:** `client/src/components/PropertyMap.tsx` (line ~166)
**Bug:** `poi.rating` from server data is interpolated into HTML without escaping.
**Fix:** Apply `escapeHtml()` to `poi.rating` the same way `poi.name` and `poi.vicinity` are escaped.

### Fix 5: sendMessage race condition
**File:** `client/src/components/rebecca/RebeccaPanel.tsx` (lines 260-358)
**Bug:** Rapid clicks (Send + chip tap) can fire two concurrent chat POSTs, corrupting conversation order.
**Fix:** Move `setLoading(true)` to the very first line of `sendMessage`, before any async work.

### Fix 6: Research pages missing ErrorBoundary
**File:** `client/src/App.tsx` (lines ~217-221, ~247)
**Bug:** `PropertyMarketResearch`, `PropertyResearchCriteria`, and `CompanyResearch` routes have no `FinancialErrorBoundary`. Crashes show full-page error.
**Fix:** Wrap these three routes in `<FinancialErrorBoundary>` the same way property detail routes are wrapped.

### Fix 7: NaN in chart tooltip
**File:** `client/src/components/dashboard/OverviewPerformanceSection.tsx` (line ~113)
**Bug:** `Number(value).toLocaleString()` shows "NaN" in tooltip when Recharts payload has a non-numeric value.
**Fix:** Add guard: `const num = Number(value); const display = Number.isFinite(num) ? num.toLocaleString("en-US") : "$0";`

### Fix 8: toLocaleString without explicit locale
**Files:** `client/src/lib/charts/DonutChart.tsx`, `BarChartInteractive.tsx`, `DonutChartInteractive.tsx`, and `components/dashboard/IncomeStatementTab.tsx`
**Bug:** `.toLocaleString()` without locale argument — on European locale machines, comma/period swap makes numbers unreadable.
**Fix:** Replace all `.toLocaleString()` with `.toLocaleString("en-US")` in financial chart contexts.

### Fix 9: export-premium localStorage bypass
**File:** `client/src/components/ExportDialog.tsx` (lines 39, 400)
**Bug:** Premium export toggle is stored in localStorage as `"export-premium": "true"`. Any user can enable it via DevTools.
**Fix:** Add a server-side check on the premium export endpoint. Client-side flag is fine for UX toggle, but the server must verify entitlement before generating the premium PDF.

### Fix 10: Admin sidebar restructuring
**Files:** `AdminSidebar.tsx`, `Admin.tsx`, `ModelDefaultsTab.tsx`
**What:** The Management Company admin group should only have: Services & Fees + Financial Statement Lines. The current "Defaults" item under Management Company contains company name, fees, staffing — those are user-configurable on `/company/assumptions`, not admin settings. See the vocabulary skill and the discussion in session memory for the full plan.
